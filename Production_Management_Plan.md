# 制造业生产管理系统 (MES/ERP) 开发计划书 (修订版)

## 1. 项目目标
构建覆盖制造全生命周期的数字化管理平台，实现数据驱动的生产协同、精准的采购周期管控以及实时的项目偏差预警。

## 2. 阶段性开发计划 (Timeline)

### 第一阶段：基础设施与核心模块开发 (W1-W2)
*   **核心任务**：
    *   数据库架构设计（PostgreSQL）及主索引优化。
    *   工单库（Work Order Library）基础功能，支持工单生命周期状态机。
    *   基础数据管理（BOM、物料主数据、供应商库）。

### 第二阶段：技术设计与工艺整合 (W3-W5)
*   **核心任务**：
    *   **图纸版本管理**：实现技术图纸的上传、存储与多版本关联。
    *   **工艺路线（Routing）**：编制标准工序、工时定额及关联技术标准。
    *   **自动流转逻辑**：图纸确认后自动激活工艺编制任务。

### 第三阶段：采购计划与供应链管理 (W6-W7)
*   **核心任务**：
    *   **采购评审流**：基于图纸需求自动生成采购计划，并支持多级在线评审。
    *   **到货周期管理**：登记生产周期与预计到货时间，系统自动校验是否满足开工条件。

### 第四阶段：生产执行与实时监控 (W8-W10)
*   **核心任务**：
    *   **任务派发系统**：将工单节点下发至具体班组。
    *   **进度实时反馈**：班组每日通过移动端或终端录入完工进度。
    *   **自动化核对**：实时计算“实际进度”与“主计划节点”的偏差天数。

### 第五阶段：数据智能与项目闭环 (W11-W12)
*   **核心任务**：
    *   **综合管理看板**：实现三色预警看板（正常/风险/延期）。
    *   **交付管理**：成品发货登记、成本自动核算、项目结案归档。

## 3. 核心机制设计

### 3.1 变更通知与强制闭环 (Change Control)
为防止变更期间出现信息断档，系统强制执行以下逻辑：
-   **锁定机制**：一旦发起技术或工艺变更，涉及到的工单将立即进入“锁定（Locked）”状态。
-   **强制确认**：在相关部门负责人（采购、生产、项目）未完成在线“变更确认”前，系统将禁用该工单后续的所有操作按钮（如：领料、汇报进度、发货）。

### 3.2 进度漏报预警 (Reporting Alert)
-   **每日巡检**：系统每日 18:00 自动检查各班组的汇报状态。
-   **自动推送**：若发现工序未按时录入进度，系统将自动生成“漏报提醒”，并实时推送到部门负责人及上级主管的终端。

## 4. 风险管理方案

| 风险类别 | 风险描述 | 应对策略 |
| :--- | :--- | :--- |
| **数据录入风险** | 一线录入不及时导致数据滞后。 | **设置“每日进度漏报”预警，系统自动根据预设时间点向上级推送催办通知。** |
| **信息沟通风险** | 变更信息未触达生产一线导致废品产生。 | **强制执行变更确认流程，在各级负责人确认前，工单处于锁定状态，无法进行下一步操作。** |
| **进度失控风险** | 项目进度偏离主计划。 | 看板自动核对逻辑：实际完工量 vs 计划节点，偏差 > 2天自动标黄，> 5天标红。 |
